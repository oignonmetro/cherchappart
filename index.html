Comment ça marche

Architecture

  • Cette page (GitHub Pages) — interface statique : critères, génération des recherches filtrées, tableau de bord des annonces, brouillons de contact.
  • GitHub Actions (le moteur) — un workflow planifié (cron) exécute le scraper toutes les quelques heures, récupère les annonces correspondant à data/criteria.json et met à jour data/listings.json. C'est ce qui permet la veille « régulière » sans serveur.

Contact automatique : le choix responsable

Envoyer des messages de façon totalement automatisée sur Leboncoin, SeLoger ou PAP enfreint leurs conditions d'utilisation et leurs protections anti-robot. ChercheAppart fait donc du contact semi-automatique : le message est pré-rédigé depuis votre modèle et ouvert en un clic (mailto ou page de l'annonce) — vous validez l'envoi. C'est fiable, conforme, et ça évite les blocages de compte.

Limites honnêtes

  • Les sites protègent leurs pages ; le scraping peut être bloqué depuis les serveurs GitHub. Le scraper est conçu pour échouer proprement et se brancher sur un proxy / une API si besoin (voir le README).
  • Le générateur d'URL est, lui, 100 % fiable — c'est le socle recommandé au quotidien.
